# Open Source Audit Project – Mozilla Firefox

This project is developed as part of the Open Source Software course with the aim of exploring how open-source systems are built, shared, and maintained. Instead of only studying theory, the project focuses on understanding a real-world open-source application in depth.

For this audit, I have selected Mozilla Firefox. Firefox is a widely used open-source web browser developed by the Mozilla Foundation. It is especially known for promoting user privacy, transparency, and control over the web experience, which makes it a strong example of open-source philosophy in practice.

## Project Overview

The project is divided into both theoretical and practical components. It includes a detailed study of Firefox’s background, its licensing model, and the ethical ideas behind open-source development. Along with this, the project also examines how Firefox functions within a Linux environment, including its installation, file structure, and system interaction.

In addition to the analysis, the project also explores the ecosystem surrounding Firefox—such as related tools, community contributions, and its role in the broader open-source world. A comparison with proprietary browsers is also included to understand the differences in flexibility, control, and development approach.

## Shell Scripts Included

To demonstrate practical Linux skills, five basic shell scripts have been created:

1. **System Identity Report**
   This script displays important system details such as kernel version, current user, uptime, and system date.

2. **FOSS Package Inspector**
   This script checks whether Firefox is installed on the system and retrieves basic package-related information.

3. **Disk and Permission Auditor**
   This script analyzes key system directories and displays their storage usage along with permissions.

4. **Log File Analyzer**
   This script reads a given log file and counts how many times a specific keyword appears.

5. **Open Source Manifesto Generator**
   This interactive script takes user input and generates a simple personalized open-source statement saved as a text file.

## Technologies Used

* Ubuntu Linux (via WSL)
* Bash Shell Scripting
* Mozilla Firefox

## Conclusion

Working on this project provided a clearer understanding of how open-source software is not just about free access, but also about collaboration, transparency, and shared knowledge. It also helped in gaining basic hands-on experience with Linux commands and shell scripting, which are essential skills for working in real-world development environments.
